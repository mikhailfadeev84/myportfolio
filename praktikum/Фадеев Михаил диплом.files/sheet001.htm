3. Что такое эквивалентность? Что такое класс эквивалентности?
Эквивалентность - это равнозначность какого-либо объекта, значения которое продукт обрабатывает одинаково.
Класс эквивалентности  - это набор каких-либо данных. с которыми тестируемый продукт должен работать одинаково.
4. Можно ли исключить проверку в середине диапазона в пользу проверок на границах, входящих в диапазон? Аргументируй свой ответ.
Нельзя, т.к. приложение может по разному обработать данные...Проверка в середине диапазона и на границах это разные проверки, которые необхтолимо провоить отдельно, чтобы исключить ошибки

Для каждого поля будем проводить позитивные и негативные проверки (спропектируем позитивные и негативные тест-кейсы), разложим требования для полей на комбинации по длине допустимого текста, по типам вводимых символов, выделим классы эквивалентности (диапазон и набор значений, граничные значения).

7. Опиши, чем чек-лист отличается от тест-кейсов. Приведи примеры, где применяют и то, и другое.
Чек-лист это список элементов, которые проверяем, состоит из элемента проверки и описаения ожидаемого результата (применяют для проверки простых элементов продукта, например тестирование верстки, кроме того такая тестовая документация составляется, в случаях ограниченного времени для проведения проверки), тест-кейс это подробное описание проверки с указанием шагов для достижения ожидаемого результата, тест-кейс состоит из обязательных и необязательных элементов. В тест-кейсах хранится информация о продукте.  Такая документация составляектся например в случае проверки работы логики продукта.  
8. Как правильно составить баг-репорт? Какие элементы баг-репорта — обязательные? Почему?
Баг-репорт - это отчет об ошибке, как правило баг-репорты заводятся в специальных инструментах (баг-трекинговые системы). Обязательными элементами являются:
- id - уникальный номер;
- заголовок (описывает суть ошибки), должен отвечать на вопросы: что, где, когда;
- шаги воспроизведения - перечень действий, для воспроизведения ошибки;
- Ожидаемый результат и фактический результат;
- Окружение (ОС, браузеры, версии приложений, в которых возникает ошибка);
- приоритет - серьезность ошибок.
Обязательные элементы необходимы, чтобы четко понимать когла и при каких обстоятельствах возникла ошибка, опредедлить срочность исправления ошибки, 		  
9. По каким правилам составляют заголовок баг-репорта? Что будет, если составить заголовок неправильно?
Заголовок составляется по правилу: Что (Что происходит) Где (В каком месте придложения, страницы) Когда (При каком действии происходит ошибка).
Заголовок описывает суть ошибки, если он будет составлен неправильно будет непонятно, какакая именно ошибка, где возникает, при каких обстоятельствах.
10. Из чего состоит клиент-серверная архитектура приложения? Кратко опиши функциональность каждого элемента.
элементами такой архитектуры являются:
- клиент (система которая связывается с сервекром, направляя запросы о необходимой для пользователя информации);
- сервер (систпема которая обрабатывает запросы клиента и формирует ответы на них);
- сеть (система связанных устройств, которая помогает клиенту и серверу обмениваться данными).
11. Опиши этапы обработки запроса после того, как в адресную строку браузера вводят URL: https://yandex.ru.
После ввода URL:
- браузер начинает искать сервер, который находится по уникальному адресу (ip-адрес). Поиск происходит сначала в кэш, а затем в DNS (распределительной системе серверов));
- браузер устанавливает соединения с сервером;
- браузер отправляет запрос, в котором просит сервер отправить данные для отображения страницы;
- сервер обрабатывает запрос и направляет ответ браузеру;
- браузер обрабатывает полученный ответ и отображает веб-страницу
12. Что такое кэш? Зачем он нужен? Какое правило нужно соблюдать при работе с кэшем в тестировании?
Кэш - это технология хранения данных веб-страниц. Данные сохраняются на устройство. Кэш нужен для быстрой загрузки страницы, часть данных загружается не с сервера, а из ранее сохраненных данных на устройстве.
Правило очистки кэша (ошибка может повториться на устройстве после исправления если не почисить кэш).
13. Ты знаешь, что есть протоколы HTTP и HTTPS.
- Чем отличаются HTTP и HTTPS? В каких случаях не стоит пользоваться HTTP?
- Из каких компонентов состоит HTTP запрос: за что отвечает каждый?
- Какие HTTP-методы ты знаешь? За что они отвечают? Приведи примеры применения разных методов.
- Что такое код HTTP-ответа? Какие коды бывают?
HTTP и HTTPS протоколы передачи данных, отличаются тем, что протокол HTTPS обеспечиает безопасную передачу данных.
Использование HTTP протоколов не безопасно, может произойти утечка данных пользователя (персональные данные, реквизиты карт, логины, пароли, данные документов).
HTTP запрос состоит из:
- стартовой строки (указывает какое действие необходимо совершить и по какому адресу направляется запрос на бэкенд);
- заголовков (передают дополнительную информацию о клипенте);
- тела сообщения (данные, которые передает фронденд).
К часто используемым методам относятся POST (отправить данные на бэкенд, например авторизация пользователя на каком-либо ресурсе), GET (запросить данные, например если необходимо получить список товаров в определенном разделе интернет-магазина, или просмотреть добавленные товары в корзину), DELETE (удалить данные на бэкенле, удалить товары из корзины в интрент-магазине), PUT (изменить данные на бэкенде, например добавить новый товар в корзину).
Код HTTP -ответа отображает статус успешно или нет обработан запрос, различают информационные коды, успешные, клиентские, серверные.

15. Какие виды мобильных приложений бывают? В чём особенность каждого?
- нативные приложения (требуют установки на устройтсво, с помощью установочного файла, могут работать офлайн);
- веб-приложения (не требуют установки на устиройство, для работы требуют интрент-соединения);
- гибридные (содержат элемиенты нативных и веб-приложений, устанавливаются отдельно на устройство, при отсутствии соединения, какая-то часть приложения остается доступна для пользователя).

18. Что такое реляционная база данных? Чем она отличается от нереляционной?
реляционная БД это совокупность таблиц с информацией, которые связаны между собой. Реляционные БД используют язык SQL (структурированных запросов). Базы данных отличаются моделями хранения данных.  В нереляционной базе данных не используется табличная схема строк и столбцов, них применяется модель хранения, оптимизированная под конкретные требования типа хранимых данных. Данные могут храниться как простые пары ""ключ — значение"", документы JSON или граф, состоящий из ребер и вершин."
19. Напиши, какие виды JOIN бывают. В чем особенность каждого?
INNER (внутреннее соединение) - соединение строк на пересечении двуx таблиц;
OUTER (внешнее соединение) выполняется тремя способами LEFT JOIN  (возвращаются все строки левой таблицы), RIGHT JOIN (возвращаются все строки правой таблицы), FULL JOIN (возвращаются строки и левой, и правой таблиц. Если у строк левой таблицы и правой таблицы выполняется условие соединения, они объединятся в одну строку. Если есть строки, у которых не выполняется условие соединения, они дополнятся NULL-значениями)
